### 2.2 表结构详细说明

#### 2.2.1 FileRecord (文件记录表)

存储文件的逻辑信息，一个文件可以有多个版本。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| Id | TEXT | PK | 主键，GUID |
| FileName | TEXT | NOT NULL, MAX 500 | 文件名 |
| CurrentVersionId | TEXT | FK → FileVersion | 当前最新版本ID |
| ContentType | TEXT | NOT NULL, MAX 200 | MIME类型 |
| VersionCount | INTEGER | NOT NULL, DEFAULT 0 | 版本数量 |
| CreatedAt | TEXT | NOT NULL | 创建时间 (UTC) |
| UpdatedAt | TEXT | NOT NULL | 更新时间 (UTC) |
| Description | TEXT | MAX 1000 | 文件描述 |
| Tags | TEXT | MAX 500 | 标签，逗号分隔 |
| IsDeleted | INTEGER | NOT NULL, DEFAULT 0 | 软删除标记 |
| DeletedAt | TEXT | NULL | 删除时间 |

**索引：**
- IX_FileRecords_FileName
- IX_FileRecords_CurrentVersionId (UNIQUE)

### 3.2 核心 API 端点

```http
Parameters:
  - file: File (required) - 上传的文件
  - folder: string (optional) - 目标文件夹
  - description: string (optional) - 文件描述

  - changeDescription: string (optional) - 版本变更说明

Response 200 OK:
{
  "message": "文件上传成功",
  "fileId": "guid",
  "versionId": "guid",
  "versionNumber": 1,
  "fileName": "example.pdf",
  "size": 1024000,
  "fileHash": "SHA256...",
  "isLatest": true
}
```

```http
POST /api/upload/init
Content-Type: application/json

Request Body:
{
  "fileName": "large-video.mp4",
  "fileSize": 1073741824,
  "fileHash": "sha256-xxx",      // 用于秒传检查
```

```http
Headers:
  Range: bytes=5242880-         // 从5MB位置继续
```

## 4. 存储提供者设计

### 4.1 IStorageProvider 接口

```csharp
public interface IStorageProvider
{
    // 基础操作
    Task<string> UploadAsync(string objectName, Stream data, string contentType, ...);
    Task DownloadAsync(string objectName, Stream destination, ...);
    Task<bool> DeleteAsync(string objectName, ...);
    Task<bool> ExistsAsync(string objectName, ...);
    Task<Stream> GetStreamAsync(string objectName, ...);
    
    // 断点续传支持
    Task<Stream> GetRangeStreamAsync(string objectName, long start, long? end, ...);
    Task<long> GetFileSizeAsync(string objectName, ...);
    Task<string> GetPresignedDownloadUrlAsync(string objectName, TimeSpan expiration, ...);
    
    string ProviderName { get; }
}
```

### 4.2 实现对比

| 功能 | LocalFileProvider | MinioStorageProvider |
|------|-------------------|---------------------|
| 存储位置 | 本地磁盘 | MinIO 服务器 |
| 路径格式 | `Storage/{folder}/{file}` | `bucket/{folder}/{file}` |
| Range 读取 | FileStream.Seek | GetObjectArgs.WithOffsetAndLength |
| 预签名URL | 返回API代理URL | 返回MinIO预签名URL |
| 适用场景 | 开发测试、无MinIO环境 | 生产环境、分布式存储 |

## 5. 断点续传机制

### 5.1 上传流程

```
1. 初始化上传
   Client ──POST /api/upload/init──► Server
   Server 检查秒传（fileHash）
   Server 创建 UploadSession
   Server ◄──返回 uploadId── Client

2. 分片上传（可并行）
   Client ──PUT /api/upload/{id}/chunks/0──► Server
   Client ──PUT /api/upload/{id}/chunks/1──► Server
   Client ──PUT /api/upload/{id}/chunks/2──► Server
   Server 保存分片到 TempUploads/
   Server 更新 UploadedChunks

3. 断点续传检查
   Client ──GET /api/upload/{id}/status──► Server
   Server ◄──返回已上传分片列表── Client
   Client 只上传缺失分片

4. 完成上传
   Client ──POST /api/upload/{id}/complete──► Server
   Server 合并所有分片
   Server 上传到 IStorageProvider
   Server 创建 FileRecord + FileVersion
   Server 清理临时文件
```

### 5.2 下载流程

```
方案1：HTTP Range
1. Client ──GET /api/download/{versionId}──► Server
   Headers: Range: bytes=0-5242879
   
2. Server 调用 IStorageProvider.GetRangeStreamAsync()
   
3. Server ◄──返回 206 Partial Content── Client
   Headers: Content-Range: bytes 0-5242879/100000000

4. 中断后续传
   Client ──GET /api/download/{versionId}──► Server
   Headers: Range: bytes=5242880-

方案2：预签名URL
1. Client ──GET /api/download/{versionId}/chunks──► Server
   
2. Server 生成多个预签名URL（每个分片一个）
   
3. Server ◄──返回URL列表── Client

4. Client 直接访问 MinIO URL 下载分片
   Client ──GET https://minio/...Range=bytes=0-5242879──► MinIO
```

| 场景 | 推荐分片大小 | 说明 |
|------|-------------|------|
| 小文件 (<100MB) | 1-2 MB | 减少HTTP请求数 |
| 中等文件 (100MB-1GB) | 5 MB | 平衡并发和开销 |
| 大文件 (>1GB) | 10-20 MB | 提高吞吐量 |
| 超大文件 (>10GB) | 50-100 MB | 减少分片数量 |

### 8.2 并发控制

| 层级 | 推荐并发数 | 说明 |
|------|-----------|------|
| 浏览器 HTTP/1.1 | 6 | 浏览器限制 |
| 浏览器 HTTP/2 | 100 | 浏览器限制 |
| 移动端 | 3-5 | 考虑网络稳定性 |
| 服务端 | 10-20 | 根据服务器配置 |

### 8.3 缓存策略

```
1. 本地缓存目录结构
   FileCache/
   └── {FileHash前2位}/
       └── {FileHash}/     # 以哈希命名的缓存文件

2. 缓存清理策略
   - 保留最新版本缓存
   - 清理7天前的旧版本缓存
   - 限制缓存总大小（如10GB）

3. 缓存校验
   - 使用 SHA256 校验文件一致性
   - 不一致时自动重新下载
```
